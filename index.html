WeJob — ระบบบริหารจัดการงานและบุคลากร

ที่มาของโครงงาน

เดิมทีการแจ้งคิวงานช่างของบริษัทต้องทำผ่าน LINE Group ส่งผลให้เกิดความสับสนในการรับงาน และอาจเกิดความผิดพลาดในการจ่ายงานได้ — โครงงานนี้จึงถูกสร้างขึ้นเพื่อแก้ปัญหาเหล่านี้ โดยทำให้ระบบสามารถดูภาพรวมงานทั้งหมดในเดือนนั้นๆได้ ทำให้วางแผนในการจ่ายงานได้อย่างมีประสิทธิภาพ และให้พนักงานสามารถดูงานทั้งหมดของตัวเองได้อย่างชัดเจน ลดการตกหล่นของงาน

ระบบทำงานอย่างไร

1
Admin — สร้างคิวงาน และจ่ายงานให้ช่าง กรอกรายละเอียดงาน ระบุลูกค้า วันที่ และสถานที่
2
ช่างปฏิบัติงาน — อัปโหลดรูปหลักฐาน และให้ลูกค้าสแกน QR Code เพื่อทำแบบสอบถามความพึงพอใจ
3
ลูกค้า — ทำแบบสอบถามความพึงพอใจ ว่ารู้สึกประทับใจในการบริการของช่างคนนี้หรือไม่
4
Manager — ตรวจสอบความเรียบร้อยของงานที่ช่างอัปโหลด และดูความคิดเห็นของลูกค้า

ความสามารถหลัก

  • ระบบจ่ายคิวงาน และติดตามสถานะงาน ผ่านปฏิทิน/ตารางงาน
  • อัปโหลดรูปหลักฐานการทำงาน ผ่านมือถือ
  • ระบบ Feedback เพื่อรับฟังความคิดเห็นจากลูกค้า ทำให้นำมาปรับปรุงแก้ไข เพื่อให้การบริการดียิ่งขึ้น

ผลลัพธ์ที่ได้

ลดงานตกหล่น · วางแผนตารางงานได้อย่างมีประสิทธิภาพ · ดูความพึงพอใจและความคิดเห็นของลูกค้า · ลูกค้าเชื่อมั่นมากขึ้นด้วยระบบสอบถามความพึงพอใจ

Pudu Fleet Management Dashboard — ระบบบริหารจัดการและติดตามสถานะหุ่นยนต์

ที่มาของโครงงาน

บริษัท นำชัยแคร์ จำกัด มีการให้บริการหุ่นยนต์จำนวนมากกระจายอยู่ตามสาขาและแบรนด์ต่างๆ แต่ยังไม่มีระบบส่วนกลางที่สามารถดูสถานะของหุ่นยนต์ทั้งหมดพร้อมกันได้ และเมื่อเกิดปัญหาขึ้น ทีมงานไม่สามารถรับรู้ได้ทันที ต้องรอให้ผู้ใช้บริการติดต่อเข้ามาก่อน — จึงทำให้การแก้ปัญหาล่าช้าและเกิดผลกระทบต่อลูกค้า

ระบบทำงานอย่างไร

1
Pudu Cloud ส่งข้อมูล — สถานะหุ่นยนต์ทุกเครื่องถูกส่งมาแบบ Real-time ผ่าน Webhook
2
Backend ประมวลผล — กรองข้อมูลที่ต้องการจาก Logs ที่ถูกส่งมาเพื่อนำไปแสดงผล และ Auto Cache Management ตอนเที่ยงคืน เพื่อรักษาเสถียรภาพ Server
3
Dashboard แสดงผล — ทีมงานเห็นสถานะหุ่นยนต์ทุกเครื่องในหน้าเดียว และระบบจะแจ้งเตือนเมื่อมีปัญหา Error
4
Log Viewer — ทีมเทคนิคสามารถตรวจสอบ Raw Logs (JSON) เพื่อวิเคราะห์ปัญหาเชิงลึก

ความสามารถหลัก

  • Webhook Endpoint รับ Real-time data จาก Pudu Cloud API ของหุ่นยนต์กว่า 400 เครื่อง
  • แสดงข้อมูลสถานะการทำงานของหุ่นยนต์ และแจ้งเตือนเมื่อหุ่นยนต์เกิด Error
  • ระบบ Memory Management Auto Cache Clear ตอนเที่ยงคืน เพื่อรักษา Server ให้เสถียรตลอด
  • Raw Logs Viewer (JSON Format) ให้ทีมเทคนิควิเคราะห์ปัญหาเชิงลึก

ผลลัพธ์ที่ได้

Monitor หุ่นยนต์กว่า 400 เครื่องในหน้าจอเดียว · รับรู้ถึงปัญหาการทำงานของหุ่นยนต์และดำเนินการแก้ไขได้ทันที ไม่ต้องรอลูกค้าแจ้ง
